## Language

**Achievement**:
A single, self-contained accomplishment โ€” the atomic unit stored in the Bank.
_Avoid_: bullet, item, entry, line

**Achievement Bank**:
The single canonical store of all Achievements. The website and every CV are views over it.
_Avoid_: database, resume data, CV store

**View**:
A rendered selection over the Bank. The website is the curated view; each tailored CV is a per-listing view.
_Avoid_: output, render

**CV Variant**:
A CV produced for a specific Job Listing by selecting and ordering a subset of the Bank โ€” the document you send for one application. ("version" is reserved for revision history, not this.)
_Avoid_: CV version, version

**Job Listing**:
The specific role/posting a CV Variant is tailored to.
_Avoid_: job, posting (when precision matters)

**Tailoring**:
Producing a CV Variant by selecting, ordering, and (via LLM, at the output layer) re-wording Achievements from the Bank for a Job Listing. Re-wording never changes the underlying facts and never mutates the Bank.
_Avoid_: customizing, tweaking

**Canonical phrasing**:
The Bank's source-of-truth wording of an Achievement (the fact + metric). The one thing Tailoring must preserve; never altered.
_Avoid_: original, base text

**Variant phrasing**:
A per-Variant re-wording of an Achievement, generated by an LLM for one Job Listing, grounded in the canonical phrasing and fact-preserving. Ephemeral and derived โ€” lives with the Variant, never written back to the Bank.
_Avoid_: rewrite, edited version

**Role**:
A position Michael held (Tastewise, Zencity, Snipe, Unit 8200). Achievements belong to exactly one Role; a Variant stays grouped by Role, reverse-chronological.
_Avoid_: job, position, company

**Identity block**:
The fixed part of every CV โ€” name, contact, education. Never tailored; the same in every Variant.
_Avoid_: header, profile (the data export `profile` carries it, but the concept is "the fixed block")

**Tag**:
A broad domain label on an Achievement (`data`, `ai`, `backend`, `frontend`, `infra`, `leadership`) used to route relevance during Tailoring. Not a specific skill or technology.
_Avoid_: skill, category, label

**Strength**:
A measure of how strong/impressive an Achievement is, used to order it within a role and to decide what to drop when a Variant runs long.
_Avoid_: priority, rank, score

**Core Achievement**:
An Achievement that appears in every CV Variant regardless of Tags โ€” the few that define who you are.
_Avoid_: pinned, featured, default
